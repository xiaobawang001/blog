# 在 GitHub 发布新文章

本站内容即 `docs/` 目录下的 Markdown 文件。**不需要写任何 frontmatter**，文件名和 h1 标题就是全部配置。

## 1. 新建一篇文章

在对应分类目录下创建 `.md` 文件，文件名即文章标题：

```
docs/
├── 使用指南/
│   └── 新文章.md       ← 文件名 = 文章标题
└── 服务器运维/
    └── docker/
        └── 新笔记.md
```

正文第一行写 `# 标题`（h1），会自动作为侧边栏显示名和页面标题。

## 2. 加入侧边栏

编辑项目根目录的 `sidebar-tree.yaml`，在对应位置添加文件名（不含 `.md`）：

```yaml
服务器运维:
  docker:
    - Docker安装
    - Docker镜像源配置
    - 新笔记           # ← 加这里
  psql:
    - psql安装
```

排序由 YAML 中的先后顺序决定，不需要 `order` 字段。

## 3. 自动构建

推送后 GitHub Actions 会自动构建部署。

本地验证：

```bash
npm run docs:build
```

## 常见问题

**Q：推送后页面没更新？**

- 打开 GitHub **Actions**，确认 workflow 成功
- 首次部署需在 **Settings → Pages → Source** 选 **GitHub Actions**

**Q：新文章不在侧栏？**

- 确认已在 `sidebar-tree.yaml` 中添加
- 文件名需与 YAML 中一致（不含 `.md`）

**Q：本地预览？**

```bash
npm run docs:dev
```
